# SOC Log Analysis Lab

## Project Overview
This project simulates a brute-force login attempt and demonstrates how to detect such activity using Linux system logs.

## What I Did
- Performed multiple failed login attempts (simulated attack)
- Used `journalctl` to analyze authentication logs
- Filtered logs to identify suspicious activity

## Key Findings
- Multiple incorrect password attempts were detected
- Repeated authentication failures occurred within a short time frame
- Logs clearly show patterns of brute-force behavior

## Tools Used
- Ubuntu Linux
- journalctl
- Terminal

## Conclusion
This project demonstrates how brute-force attacks can be identified through log analysis and highlights basic SOC analyst skills.
